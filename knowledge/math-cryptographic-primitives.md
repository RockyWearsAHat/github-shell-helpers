# Cryptographic Primitives — Mathematical Foundations

## Overview

Cryptographic primitives are mathematical constructs that form the building blocks of secure communication systems. Unlike applied cryptography (which focuses on protocols like TLS or PGP), this domain addresses the pure mathematics: the hard problems that make certain constructions secure, the algebraic structures they're built on, and why specific mathematical properties can be relied upon for confidentiality and authenticity.

## Modular Arithmetic and Finite Fields

### Modular Arithmetic Fundamentals

**Modular arithmetic** operates on equivalence classes of integers under a fixed modulus $m$. Two integers $a$ and $b$ are congruent modulo $m$ (written $a \equiv b \pmod{m}$) if $m$ divides $(a - b)$.

Core properties that cryptography exploits:
- **Cyclic structure**: The set $\mathbb{Z}_m = \{0, 1, ..., m-1\}$ under addition forms a cyclic group
- **Multiplicative group**: For prime $p$, the non-zero elements $\mathbb{Z}_p^*$ form a group under multiplication with order $p-1$
- **Carmichael's theorem**: For any $a$ coprime to $n$, $a^{\lambda(n)} \equiv 1 \pmod{n}$ where $\lambda$ is Carmichael's lambda function

This structure is foundational to RSA: security relies on the gap between computing $\lambda(n)$ (equivalent to factoring $n$) and verifying that $a^d \equiv b \pmod{n}$.

### Finite Fields

**Finite fields** $\mathbb{F}_q$ contain exactly $q$ elements (always $q = p^k$ for prime $p$ and positive integer $k$). The most common are prime fields $\mathbb{F}_p$ for large primes $p$.

Key properties:
- Both addition and multiplication form groups
- Every non-zero element has a multiplicative inverse
- **Multiplicative group structure**: $\mathbb{F}_p^*$ is cyclic of order $p-1$
- **Primitive roots (generators)**: Elements $g$ such that $\{g^1, g^2, ..., g^{p-1}\}$ generates all non-zero elements

Cryptographic applications rely heavily on working in $\mathbb{F}_p$: exponentiation is tractable, but finding discrete logarithms is hard.

## Elliptic Curve Mathematics

### Curve Equations and Point Addition

An **elliptic curve** over $\mathbb{F}_p$ typically takes the Weierstrass form:
$$y^2 \equiv x^3 + ax + b \pmod{p}$$

where $4a^3 + 27b^2 \not\equiv 0 \pmod{p}$ (non-singular condition).

The set of points $(x, y) \in \mathbb{F}_p^2$ satisfying this equation, plus a special "point at infinity" $\mathcal{O}$, forms an abelian group under a geometric operation: **point addition**.

For points $P = (x_1, y_1)$ and $Q = (x_2, y_2)$:
- If $P = Q$: the line through $P$ is tangent to the curve; the third intersection yields $P + P = 2P$
- If $P \neq Q$: the line through them intersects the curve at a third point; reflecting across the $x$-axis gives $P + Q$
- Identity element: $\mathcal{O}$ (point at infinity)

This geometric interpretation provides the **group law**, allowing **scalar multiplication**: $[k]P = P + P + ... + P$ ($k$ times) computed via repeated doubling in $O(\log k)$ operations.

### Discrete Logarithm Problem (ECDLP)

Given $P, Q$ on an elliptic curve where $Q = [k]P$ for unknown scalar $k$, the **Elliptic Curve Discrete Logarithm Problem** asks: compute $k$.

Difficulty estimates:
- For a curve of order $n$, the best known algorithm (Pollard's rho) requires $O(\sqrt{n})$ elliptic curve point additions
- For $n \approx 2^{256}$, this is computationally infeasible (~$2^{128}$ operations)
- Contrast this with integer factorization or discrete log in multiplicative groups, where subexponential algorithms (number field sieve) apply

This exponential-time hardness (no subexponential algorithm known) is why ECC permits smaller key sizes than RSA or classical DLP for equivalent security levels.

## Group Theory for Cryptography

### Group Structure and Order

A **group** $(G, \cdot)$ is a set with an associative binary operation where every element has an inverse and an identity element exists.

For cryptography:
- **Order of element**: smallest positive integer $k$ such that $g^k = e$ (identity)
- **Cyclic group**: can be generated by a single element $g$; all finite cyclic groups of order $n$ are isomorphic to $\mathbb{Z}_n$
- **Subgroup**: a subset forming a group under the same operation; **Lagrange's theorem** states the order of a subgroup divides the order of the group

Cryptographic constructions leverage this: Diffie-Hellman and discrete log security depend on working in a large cyclic subgroup of $\mathbb{F}_p^*$, avoiding small-order subgroups that would weaken security (small subgroup attacks).

### Generators and Orders

A **generator** (primitive root) of $\mathbb{F}_p^*$ is an element of order $p-1$. For a prime $p$, exactly $\phi(p-1)$ generators exist (where $\phi$ is Euler's totient function).

Safe primes $p = 2q + 1$ (where $q$ is also prime) ensure that the multiplicative group has order $2q$, providing two large prime factors and avoiding small-order elements.

## Lattice Problems and Post-Quantum Cryptography

### The Shortest Vector Problem (SVP)

A **lattice** $\mathcal{L}$ is a discrete additive subgroup of $\mathbb{R}^n$, generated by a basis matrix $B$ where $\mathcal{L} = \{Bx : x \in \mathbb{Z}^m\}$.

The **Shortest Vector Problem** asks: given a basis for $\mathcal{L}$, find a non-zero vector $v \in \mathcal{L}$ with minimal Euclidean norm.

Hardness properties:
- SVP is NP-hard in the worst case
- For random lattices, no efficient algorithm is known even in approximation versions
- Quantum computers offer no known subexponential speedup (unlike factorization or discrete log)

This quantum-resistant hardness makes lattice problems appealing for post-quantum cryptography.

### Learning with Errors (LWE)

The **Learning with Errors** problem: Given samples $(a_i, b_i)$ where $a_i \in \mathbb{Z}_q^n$ and $b_i = \langle a_i, s \rangle + e_i \pmod{q}$ (with $s \in \mathbb{Z}_q^n$ secret and $e_i$ small error), recover $s$.

Properties:
- With no error, this is linear algebra (easy via Gaussian elimination)
- With small error, the problem becomes hard: errors mask the secret structure
- LWE (over polynomial rings, "Ring-LWE") is believed to be as hard as SVP
- Security is based on averagecase hardness, unlike SVP's worst-case hardness

LWE forms the basis of lattice-based encryption (e.g., Kyber in NIST's post-quantum standardization).

## Pairing-Based Cryptography

### Bilinear Pairings

A **bilinear pairing** on elliptic curve groups is a map $e: G_1 \times G_2 \to G_T$ where:
- $G_1, G_2$ are additive groups (points on elliptic curves or related structures)
- $G_T$ is a multiplicative group (subgroup of a finite field extension)
- **Bilinearity**: $e([a]P, [b]Q) = e(P, Q)^{ab}$ for all $P, G_1$, $Q \in G_2$, scalars $a, b$

Common pairings: **Weil pairing** and **Tate pairing** (computed via Miller's algorithm).

Applications that became possible:
- **Identity-based encryption**: public key can be an arbitrary string (user email)
- **Attribute-based encryption**: fine-grained access control via attributes
- **Short signature schemes**: signatures shorter than RSA at equivalent security
- **Threshold cryptography**: secret sharing with cryptographic properties

The pairing structure reduces certain discrete log instances to others: the **pairing problem** relates to finding $e(P, Q)^x$ without knowing $x$, enabling entirely new protocol designs.

## Zero-Knowledge Proofs — Mathematical Core

### Interactive ZK Proofs

A **zero-knowledge proof** is an interactive protocol where a **prover** convinces a **verifier** of a statement's truth without revealing why it's true.

Formal properties:
- **Completeness**: if the statement is true, an honest prover always convinces the verifier
- **Soundness**: if false, no prover (except with negligible probability) convinces the verifier
- **Zero-knowledge**: verifier learns nothing except the truth of the statement (can simulate the transcript)

Mathematical instantiations rely on hard problems:
- Fiat-Shamir: proving knowledge of a square root modulo $n$ (relies on factorization hardness)
- Guillou-Quisquater: generalizes Fiat-Shamir using RSA structure

### Non-Interactive ZK and Pairings

Non-interactive zero-knowledge proofs become possible when using:
- **Pairing-based constructions**: Fiat-Shamir heuristic applied to pairings (Common Reference String model)
- **Polynomial commitments**: commit to a polynomial, then prove evaluations via elliptic curves
- **SNARKs**: Succinct Non-Interactive Arguments of Knowledge, combining pairings with polynomial constraints

These enable **circuit satisfiability proofs**: prover proves correctness of a computation without revealing inputs, critical for privacy-preserving applications like anonymous cryptocurrencies and confidential transactions.

## Cryptographic Assumptions and Their Status

| Assumption | Instance | Hardness | Quantum | Key Size |
|-----------|----------|----------|---------|----------|
| Integer Factorization | RSA-2048 | Subexponential (GNFS) | Broken | 2048 bits |
| Discrete Log (multiplicative) | $\mathbb{F}_p^*$, $p \approx 2^{2048}$ | Subexponential | Broken | 2048 bits |
| Elliptic Curve DLP | $\mathbb{F}_p$ subgroup, $\approx 2^{256}$ order | Exponential (best: Pollard rho) | Resists | 256 bits |
| Lattice-based (SVP/LWE) | $n = 1024$ dimension, $q \approx 2^{30}$ | No subexponential known | Resists | 1024 bits ~ 2048 bits |
| Pairing (CDH in $G_T$) | $r$-bit prime order | Depends on pairing type | No known speedup | 512-768 bits |

## Connections and Trade-offs

**Hard vs. easy directions**: cryptographic hardness is asymmetric. Hashing and digital signatures rely on computational hardness *in the forward direction* (computing is hard; verifying is easy), while encryption relies on hardness in both directions (should not leak partial information).

**Parameter choices matter**: Using a 2048-bit modulus for classical DLP provides only 112 bits of security (subexponential attacks). Elliptic curves with 256-bit coordinates provide ~128 bits. Lattice-based schemes at 1024-dimensional lattices and quantum security require different parameter selection than their post-quantum safety margins assume.

**Hybrid approaches**: Critical systems use multiple hardness assumptions. TLS 1.3 uses both elliptic curves (ECDH for key agreement) and RSA/ECDSA (for signatures), accepting that breaking one does not compromise the other.

The field remains deeply connected to number theory, algebra, and complexity theory; advances in any discipline (e.g., faster integer factorization algorithms, new lattice reduction techniques, or breakthroughs in pairing computation) immediately impact cryptographic practice.
