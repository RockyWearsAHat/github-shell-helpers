## Highlights

- **Branch sessions are working.** Copilot agents can now create isolated branches via `branch_session_start` / `branch_session_end`. Each chat session gets its own branch, and switching between chats automatically switches the visible branch in the main repo. The worktree mechanics are invisible to the user — `git branch`, `git status`, and the VS Code Explorer all reflect the active chat's branch.
- Eliminated JSONL dependency from the worktree binding and focus system. Bindings are now keyed directly by worktree path, and chat focus is driven by VS Code tab events instead of file-system polling. JSONL watcher is retained only for sidebar activity display.
- Removed ~225 lines of dead code (15+ functions) from the extension, including the entire session-ID resolution chain (`findBestSessionIdByTitle`, `findActiveSessionId`, `getActiveChatTabContext`, etc.).
- Cleaned up legacy workspace-root swapping code and inline `require()` calls.
- Updated branch-lifecycle instructions: branch sessions are now the preferred workflow when enabled (previously marked as "only when explicitly needed").
- Added Branch Workflows section to README documenting the feature for users.

## Breaking Changes

- `worktreeBindings` globalState key migrated to `worktreeBindings.v2` (keyed by worktree path instead of session ID). Old bindings are not migrated — active sessions from v0.3.4 will need to be restarted.
