## Fix

- **MCP server now starts for fresh agents/subagents.** Claude Code could be
  left with the `helpers` server registered as the retired `helpers-mcp` Node
  launcher shim (which execs a `helpers-serverd.js` that no longer ships). That
  only "connected" while a warm daemon lingered; every fresh process — a spawned
  agent/subagent, or any connect after the daemon died — failed with
  `MODULE_NOT_FOUND`. `helpers doctor`/`status` now verify the registration
  actually runs the native `helpers-native mcp` server (rejecting the dead shim)
  and **self-heal** a missing/stale registration. `helpers update` and the npm
  postinstall re-point the registration after a binary-only upgrade, and the
  installer no longer ships the dead Node shim.

## Change

- **`helpers grade` (cs-grade) is now language-aware.** The grader was hardwired
  to Java, so a non-Java project found zero Java and scored an automatic F that
  reflected a language mismatch, not the code. The CS2420/CS3500 principles it
  grades (design/structure, data structures, tests, documentation, style, build,
  abstraction) are language-agnostic; only the syntactic signals are not. A new
  per-language profile detects the dominant language — **Java, Rust, Python,
  TypeScript/JavaScript, Go, C/C++**, plus a generic fallback — and supplies the
  right signal patterns and report wording. The report header and `--json`
  payload now name the detected language. Java grading is unchanged (the Java
  profile reproduces the original patterns and wording exactly); the
  "Documentation & Javadoc" category is renamed "Documentation & comments".
