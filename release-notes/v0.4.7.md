## Release Scope

Adds an end-to-end **install-test CI** that proves every install method provisions
working tools with no build toolchain, and fixes a Windows prebuilt-download bug it
surfaced.

## Fixes

- **Windows `helpers build` could not unpack the prebuilt.** git-bash's GNU `tar` read
  the Windows `C:\…` archive path as a remote `host:path` ("Cannot connect to C"), so the
  prebuilt extraction failed and it fell back to a (missing) source build. Extraction now
  uses a relative filename from a temp cwd — portable across macOS/Linux/Windows.
- **`helpers build` now explains why a prebuilt wasn't used** before falling back to a
  source build (no prebuilt for platform / curl|tar missing / download or extract failure /
  binary missing), instead of failing silently.

## CI

- New **install-test** workflow: installs Helpers via the curl installer, portable tarball,
  npm, `.deb`, and `.pkg` across ubuntu/macOS/Windows with `cargo`/`rustc`/`cc` hidden, and
  asserts the native tools are provisioned by **downloading the prebuilt** (`> 0` tools, no
  compilation). It also runs `helpers-native` standalone to confirm the native tools carry
  no Node dependency. Runs on PRs touching install/packaging paths, and weekly.

> Node remains the runtime for the `helpers` CLI and the MCP server (the latter hosts the
> browser-based `search_web`/`scrape_webpage`); the heavy native tool binary is Node-free.
