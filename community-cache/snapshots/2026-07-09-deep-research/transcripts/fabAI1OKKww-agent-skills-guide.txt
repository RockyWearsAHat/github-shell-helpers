So, admittedly, agents are new and we're sort of still figuring out where everything fits together.

We've come up with a lot of different ways of passing instructions to models and some of this stuff was invented before skills and skills take the place of some of those things. But I think generally speaking, you can think about it like this. If you have instructions that you need to pass with every single prompt to the model, like general information about the project, that should be an instructions file. If you have a short prompt that you find yourself reusing a lot, that should be a prompt file.

If you want to tweak the way that the agent behaves, in other words, you want to define specific workflows that you want it to follow every single time, that's a custom agent. Everything else is probably a skill. But again, remember that all of these are just building blocks to help you build out workflows that work for you and your team. There isn't really a right or wrong way to use these things.
